# Plan a Project or Initiative

**Category:** Productivity
**Tags:** project-planning, milestones, timeline, execution

## What it does

Turns a goal into a clear, actionable plan — phases, milestones, owners, dependencies, and a realistic timeline.

## Prompt

```text
You are an experienced program manager. Turn the goal below into a clear plan.

Produce:
1. **Goal & success criteria** — restate the objective and how we'll know it's done.
2. **Phases** — break the work into 3-5 logical phases.
3. **Milestones & deliverables** — key checkpoints for each phase.
4. **Tasks** — the main tasks under each phase, with a suggested owner type and
   rough effort.
5. **Dependencies & risks** — what could block progress and how to de-risk it.
6. **Timeline** — a realistic sequence; flag anything that can run in parallel.

Rules:
- Keep it practical and right-sized for the scope — don't over-engineer.
- Call out assumptions and anything you'd need to confirm before starting.

The goal / project: {{what you're trying to accomplish}}
Timeframe: {{deadline or target window}}
Resources / team: {{who or what you have to work with}}
Constraints: {{budget, scope, or other limits}}
```

## Example

**Input:** A goal to launch a new internal enablement program in one quarter.

**Output:** Success criteria, phases, milestones, tasks with owners, risks, and a sequenced timeline.

## Tips

- Give it your real timeframe and constraints — the plan gets far more realistic.
- Ask it to convert the output into a checklist or table if you want to track it.
